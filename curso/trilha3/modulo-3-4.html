📝 generate.ts — chamada à API Anthropic

const message = await client.messages.create({ model: "claude-sonnet-4-6", max_tokens: 4096, temperature: 0.7, system: [{ type: "text", text: SYSTEM_PROMPT, // ~3000 tokens, 8 camadas cache_control: { type: "ephemeral" } }], tools: [TOOL_SCHEMA], tool_choice: { type: "tool", name: "emit_seedance_prompt" }, messages: [{ role: "user", content: `Scene description: ${scene}` }] });

Trade-offs dos parâmetros

  • • model: "claude-sonnet-4-6" — balance de qualidade/custo. Opus é ~3x mais caro sem ganho proporcional para esse caso.
  • • temperature: 0.7 — variabilidade nos detalhes, classificação estável. Subir para 1.0 degrada consistência; baixar para 0.3 deixa repetitivo.
  • • max_tokens: 4096 — output típico ~1500 tokens, folga para casos extremos.
  • • cache_control: ephemeral — cache de 5 min. Economiza ~89% do custo do system prompt após warmup.
  • • tool_choice: forced — bloqueia injection. Não opcional.

🔐 check-email.ts — HMAC allowlist

Resolve o problema do original (email puro). Em vez de "cliente manda email, servidor confia", o servidor emite um token assinado com HMAC-SHA256 + expiração.

// Se email está no allowlist: const expiresAt = Date.now() + 30 * 24 * 60 * 60 * 1000; const payload = `${email}.${expiresAt}`; const signature = createHmac("sha256", PRO_TOKEN_SECRET) .update(payload) .digest("hex"); const token = `${Buffer.from(payload).toString("base64url")}.${signature}`; return { pro: true, token, expiresAt };

💡 Por que HMAC resolve

O cliente guarda o token. Cada chamada ao /api/generate re-valida a assinatura com a secret server-side. Tamperproof: trocar o email ou a data no cliente quebra a assinatura. Expiração embutida protege contra replay de longo prazo.

Custo real por geração

  • • Primeira chamada (cold cache): ~$0.00915 (paga 100% dos 3000 tokens de system)
  • • Chamadas subsequentes (warm): ~$0.024 inclui o custo extra dos tokens variáveis
  • • Economia do prompt cache: ~89% nos tokens de input após warmup
  • • Output típico: ~1500 tokens → custo fixo independente do cache

Custo $0.024 por prompt gerado. Cobrando $0.50 por prompt (o produto estudado cobra via Skool ~$X/mês com limite de 100), a margem é 20x. Espaço para cobrir Netlify, marketing e lucro tranquilamente.

➕ Extensões recomendadas

  • • Streaming response: mostrar tokens chegando em tempo real melhora UX percebida ~40%. Use Server-Sent Events (SSE) no Netlify Function.
  • • Webhook para fal.ai: depois de gerar o prompt, chamar automaticamente fal.ai com ele — pipeline "ideia → prompt → vídeo" em uma chamada só.
  • • Dashboard de custo: usar Anthropic Usage API para mostrar quanto cada geração custou. Permite otimizar temperature/max_tokens baseado em dados reais.
  • • Cache Redis compartilhado: para instâncias multi-region, rate limiting em memória não serve. Usar Upstash Redis como bucket.
  • • Analytics anônimo: contar quais presets mais são acionados. Permite descobrir nichos emergentes e calibrar a taxonomia.
